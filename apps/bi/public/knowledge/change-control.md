# 变更控制（一页纸）

**版本**: v0.1  
**日期**: 2026-07-11  
**适用**: 全体 13 人 + Agent  
**原则**: 先收声、再评估、管理者拍板；禁止口头改范围

---

## 1. 什么算「变更」

凡是会改下面任一者，都算变更（不只是写代码）：

| 类型 | 例子 |
|------|------|
| 范围 | 新增缺陷类型、新增必须交付物、砍掉 BI/演示 |
| 进度 | 里程碑日期平移 ≥ 1 周 |
| 成本 | 单笔或累计预计超账户预算，或动用 **reserve** |
| 质量目标 | 改 DR/FPR/mAP 门禁 |
| 技术主路径 | 换在线主模型族、取消某层能力 |

**不算变更（走意见箱即可）**：措辞优化、文档笔误、个人学习笔记、未承诺的探索实验（不占关键路径）。

---

## 2. 流程（四步）

```
提出 → 登记 → 评估 → 决策 → 同步
```

1. **提出**  
   - 队员：写入 `docs/feedback-inbox/inbox/`（用需求模板，`type: requirement`）  
   - 或在周会口头提出后 **24h 内补文件**（无文件视为未提出）

2. **登记**  
   - PM/Agent 在周 digest 或本页附录「变更台账」记一条：`CR-YYYYMMDD-##`

3. **评估**（必须写清）  
   - 对范围 / 进度 / M币 / 质量 / 风险的影响  
   - 建议：采纳 / 延期 / 拒绝 / 需张老师知情

4. **决策**  
   - **课题管理者**拍板；动用 reserve 或改 M1–M6 目标须管理者确认  
   - 重大范围/验收变更：建议同步张老师后再锁

5. **同步**  
   - 改 `requirements.md` / 章程 / 里程碑 / M币预算（如需要）  
   - 原意见文件移入 `feedback-inbox/processed/`  
   - 周报写一句「本周变更」

---

## 3. 决策权限

| 事项 | 谁说了算 |
|------|----------|
| 文档笔误、模板微调 | 任何人改，PR/提交说明即可 |
| 任务分工微调（不改里程碑） | 课题管理者或指定 PM 支持 |
| 范围/验收/里程碑/总预算 | **课题管理者** |
| 动用 reserve | **课题管理者** |
| 对外口径（对张老师/企业） | 课题管理者 |

Agent **只有建议权**，不得因单条意见自动改章程。

---

## 4. 紧急通道

若阻塞产线演示或结题硬门禁：

- 口头可先定方向，**当日**补 `CR` 记录与意见箱文件  
- 仍须管理者确认后改正式文档
