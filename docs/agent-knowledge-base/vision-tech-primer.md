---

## 1. 机器视觉检测在干什么

输入一张（或一串）图像 → 判断 **有没有缺陷 / 在哪 / 什么类型 / 有多大** → 输出报警或记录。

和手机拍照识猫不同：工业场景要 **稳、快、可解释、可抽检**，还要扛光照和产线速度。

---

## 2. 四类常见技术（先认门牌）

| 类型 | 一句话 | 本项目里 |
|------|--------|----------|
| 异常检测 | 只学「正常长什么样」，偏离即异常 | 冷启动 L1（如 PatchCore） |
| 目标检测 | 输出框 + 类别 | 有标注后的主检测路径 |
| 语义/实例分割 | 输出像素级掩码 | 框不准的难例（如 SAM2） |
| 多模态 / VLM | 看图+文本，可辅助描述与预标注 | **离线**标注飞轮，不当裁判 |

开放词汇检测（如 Grounding DINO）：用文字提示找物体，少样本时好用，但往往偏重，适合近线而非唯一在线模型。

## 3. 本课题架构怎么记（考试版）

```
在线：尽量轻 → 先筛异常 / 轻量检测 → 报警
近线：定位类型、难例分割
离线：VLM 预标注 → 人复核 → 再训练
```

完整决策：`docs/02-tech-research/ADR-001-four-layer-vision-stack.md`

## 4. 你会听到的指标

| 指标 | 人话 | 我们的目标感 |
|------|------|--------------|
| DR 检出率 | 真缺陷被抓到的比例 | ≥ 95%（冲刺目标） |
| FPR 误检率 | 好品被冤枉的比例 | ≤ 5% |
| mAP | 检测框准不准的综合分 | 里程碑门禁用 |
| FPS / 时延 | 够不够产线节拍 | 工程约束 |

没有数据时，BI 上这些指标会 **灰显**，防止假达标。

---

## 5. 光学为什么也重要（一句话）

算法吃的是相机+镜头+光源「喂」出来的图。光照乱、模糊、反光，再强的模型也稳不住——所以后期有光学/硬件角色，不是纯软件题。

---

## 6. 新队员参与技术时的规矩

1. 先问：这是 **在线 / 近线 / 离线** 哪条路径？  
2. 实验必须记版本（数据版本 + 超参），模板在 `templates/experiment-log.md`  
3. 想换主路径 → **变更控制**，不是群里一句「我们改用某某大模型」  
4. 算力消耗记 **M币**（compute 账户）
