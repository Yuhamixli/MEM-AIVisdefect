2. 标注工具
2.1 现状：LabelImg（已可用，沿用现有操作文档）
环境：conda activate huaduo；pip install labelimg；终端执行 labelImg 启动。
操作：Open Dir 指定图片目录 → Change Save Dir 指定标注文件保存目录 → 逐张标注。快捷键：D = 下一张；W = 开始标注（鼠标拖拽画框，选类名 slug）。
产出：矩形框标签文件，保存目录与 §3 目录结构对应；labels 初始为空，标注后自动生成同名标签文件。
标签格式【建议】：保存格式统一选 YOLO（.txt，类别序号 + 归一化框坐标），与 ultralytics 训练管线直接对接；类别序号与 §1 slug 表的对应关系写入 classes.txt 并随数据集版本归档，禁止各标各的格式（PascalVOC XML 仅在外部协作时作中间格式，入库前转换）。
局限：纯人工框选，产能受人时约束；无预标注能力，与 L4 飞轮不衔接。
2.2 推荐升级：X-AnyLabeling【建议，双轨评估后切换】
理由：支持 AI 预标注（可挂检测模型 / VLM），契合 ADR-001 的 L4「VLM 飞轮」：预标注 → 人工复核确认 → 训练集升级，把标注人力从「画框」转移到「判框」。
人机分工：AI 出预标注框 → 人工逐图复核修正 → 抽检合格（§5.1）后入训练集；复核不通过整批回流。
切换纪律：X-AnyLabeling 与 LabelImg 双轨并行约一周，对比产能（图/人时）与错标率后再决定是否切换；切换前 LabelImg 保留为兜底工具。预标注模型的版本与复核口径记录在数据集变更日志（§6.2）。
激励差异：预标注 + 复核按降权入账，见 §5.3。
3. 目录与数据划分
3.1 目录结构（沿用现有约定）
dataset/
  images/
    train/  val/  test/
  labels/
    train/  val/  test/
images 下按 7:2:1 将图片样本分到 train / val / test；labels 结构镜像，标注后生成同名标签文件。
3.2 划分纪律（防泄漏，评测可信的前提）
拆分单位：按 physical_piece_id（物理件）+ 批次拆分，禁止同一物理件的不同分段图同时落入 train 与 test——同件纹理高度相似会造成虚高指标（评测设计参照 Tabernik 2020 按件分组）。
50 件金标准盲测集：按 physical_piece_id / 批次抽取 50 件，冻结并独立存放（建议 dataset_golden50/，与训练集物理隔离），严禁以任何形式进入训练、验证或调参；盲测集标签双人精标 + 仲裁后作为真值基准。
抽样代表性：金标准集须覆盖先三类缺陷件与 OK 件，配比随样本盘点后定【待补，关联 50 件评测方案】。
3.3 命名规范
图像文件：{piece_id}_{batch}_{surface}_{seq}.jpg（与《图像采集方案》§7.2 采集命名一致）；标签文件同名（扩展名随工具格式）。
surface（检测面）取值集合待现场四方位编号冻结【未决，关联 OD-COM-03/04】，冻结前字符串透传不解析。
4. 标注质量标准
4.1 通用框规则（矩形框为主）
贴边：框贴缺陷外缘，不截断、不带入大片背景；
不合并：多个独立缺陷各自一框，气泡群禁止合并成大框；
分段：超长划伤 / 裂纹允许分段框取，备注同一缺陷编号；
斜向：斜向裂纹允许斜外接框或分段框。
4.2 边界样本规则
发丝级裂纹、针尖级气孔、边界弥散发白等 → 登记为边界样本，一律双人复核后再入库；
边界弥散样本打「边界弥散」标记，供训练时降权或复核（对齐定义卡 D06）；
小于最小可检尺寸的缺陷【尺寸阈值待标定，随 OD-COM-16 分辨率定版】：照常标注但打「亚检」标记，是否计入验收指标【建议不计入，待评测口径会确认】。
4.3 易混淆对处理（四对，与定义卡 §3 一致）


易混淆对

区分提示

标注处理动作

划伤 vs 裂纹

划伤偏机械擦痕；裂纹偏开裂、可能有深度感

拿不准 → 挂起进仲裁队列，禁止猜标

发白 vs 脏污

发白多为材质/浸渍外观；脏污为附着污染

可尝试清洁验证区分；擦拭可除的备注「可清洁」

气泡 vs 异物

气泡偏本体空腔/鼓包；异物为外来物、边缘更锐或材质不同

来源可辨时（如脱模布屑）备注记录

缺纱 vs 发白

缺纱见纱束缺失几何；发白以颜色/光泽为主

重叠样本标主特征并备注说明
仲裁队列由 data + struct-eng 共判，结论回写定义卡易混淆说明。
4.4 与允收标准的一致性
「算不算缺陷」以《七类缺陷定义卡》允收标准为准；允收数值留白，待 struct-eng 共签【未决@OD-COM-11】（骨架 + 配图 ≤2026-07-30；阈值随 50 件金标准复核后冻结）。
共签前临时口径【默认假设】：现场典型缺陷（8~10mm）全部标注；更小者按 §4.2 边界样本管理。
定义卡配图完成后，标注员判标以「良品对照图 + 典型缺陷图」为基准，新人上岗先学定义卡再上手。
5. 抽检制度与产能管理
5.0 标注作业流程（七步）
领任务：按批次领取标注包（含图像清单与批次号），确认所用数据集版本；
学定义卡：通读《七类缺陷定义卡》与本文 §4，重点看良品对照图与易混淆对；
试标：同批先试标 20 张，由 data 负责人确认框规则掌握情况；
双标考核：进入双标子集（§5.2），一致性达标后转为正式标注；
正式标注：按 §4 框规则标注，拿不准的一律挂起进仲裁队列，不猜标；
抽检：提交批次按 §5.1 走 10% 抽检，不合格按错标回流处理；
入库：合格批次并入对应 split，登记数据集变更日志与 M币台账（§5.3、§6.2）。
5.1 抽检
比例：每批 10% 抽检（随机抽样 + 边界样本必检）；
错标回流：错标样本回流重标；同标注员同批次错标率 > 阈值【建议 5%，实测后定】→ 该批全量复检并记录；
漏检闭环：评测 / 试跑中发现的漏检样本 → 回看原图 → 重标 → 补入训练集 → 再训练，全链路记录进数据集变更日志（§6.2），这是查全率 ≥80% 门禁的主要改进机制。
5.2 一致性度量【建议】
设约 5% 双标子集（两人独立标注同一批图），计算标注一致性（框 IoU、类一致率），作为标注员上岗考核与复核依据；阈值随首批数据实测后定。

6. 数据版本管理
6.1 raw-data 入库版本化
原始图像先入 raw-data（只增不改），按采集批次建目录：raw-data/{date}_{batch}_{lighting_cond}/；
入库登记：采集批次、相机 / 光源配置、标定版本、件数、操作人（与《图像采集方案》§7.2 采集合规检查单对接）；
数据集版本号：dataset-vX.Y——数据内容变更（新增批次、迁移、补标）升 X，修正性变更升 Y。
6.2 数据集变更记录
每次变更写入变更日志：版本 / 日期 / 变更内容（新增批次、旧名迁移、错标回流、漏检补标）/ 影响范围（涉及 split 与件数）/ 操作人。旧名迁移必须逐张记录（§1 映射表），金标准盲测集一经冻结只读、不进变更。
6.3 实验绑定数据版本
每次训练 / 评测实验记录四元组：数据版本 + 代码 commit + 模型版本 + 指标，四者绑定，模型版本号写入结构化输出的 model_version 字段；
50 件盲测指标只在冻结版数据上产生；报告逐类指标 + 置信区间（样本量局限须如实呈现：如 40/50 的 95% Wilson 区间为 67%~88.8%，过门禁不等于证量产）。
6.4 权限与脱敏
对外一律「V公司」；raw-data 中含现场信息的材料入飞书前须过脱敏检查（按《资料入库规范》执行）。
