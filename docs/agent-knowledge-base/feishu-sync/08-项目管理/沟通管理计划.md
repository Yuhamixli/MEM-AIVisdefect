3. 升级路径与变更控制衔接
组内讨论（工作流群/技术小群，随时）
  → 周六周会（上会讨论，状态转 discussing）
    → 范汝杰裁决（关单：转 closed，填决议摘要+日期，链纪要）
      → 需张老师/课程侧答疑的，列入答疑清单统一提问
升级触发：逾期 7 天未关闭 / 跨工作流无法收敛 / 影响 P0 硬交付。
变更控制衔接（铁律）：任何口头意见（含老师、导师、群聊）不直接改需求；范围、指标、节点变化一律先登记 OD→上会→裁决→改文档。任务书硬指标与内部冲刺指标冲突时以任务书为准，内部冲刺指标标"软"（如"效率提升≥30%"）。
对外口径：未经范汝杰确认的进展不得写入双周报；未达门禁的技术结论不得对外承诺。

4. 会议纪要与留痕
纪要模板（固定五段）：① 时间/参会/缺席；② 决议（已决议，逐条，含 Owner 与期限）；③ 未决（转 OD 编号）；④ 风险与触发条件变化；⑤ 下次会议时间与待办清单。
归档时限与位置：纪要48 小时内归档至仓库 docs/05-reporting/meetings/，并同步飞书「08-项目管理」目录；closed 的 OD 当周同步 BI 知识库。
飞书纪律（D-06）：可放工艺总结、代码库入口、客户现场图、全套项目文档；禁止放 SOUL.md/AGENTS.md 等个人 Agent 灵魂文件；双周报按 §2 腾讯文档→飞书双轨。
留痕边界：涉及 V公司现场与工艺的影像资料，归档前确认脱敏；对外发布一律走执行组审核。

5. 信息辐射器与异步沟通
BI 管理看板（apps/bi）：进度、M币消耗、风险状态周更，作为全员异步对齐的第一入口；例会开场先看板后发言，减少口头汇报时长。
工作流群分工：主群只发三类消息——决议通报、周会/双周报提醒、阻塞求助；技术讨论沉到算法/光学小群；文件一律发链接不发原件，以仓库为单一事实源。
异步规则：工作日消息 24h 内响应；阻塞类消息 @范汝杰 后 12h 内必须给出"受理/转交/上会"三种答复之一；紧急事项（影响当周门禁）可直接电话，事后 24h 内补文字留痕。
新成员/新信息入口：onboarding 三件套=共享简报+ADR-001+本计划，读完方可进技术小群发言定案（对应风险 R-07 监控）。
