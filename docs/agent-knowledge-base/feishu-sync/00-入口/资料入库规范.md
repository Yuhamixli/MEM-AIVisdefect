1. 命名规则
统一格式：NN_主题_版本_日期.扩展名
NN：两位文件夹序号，与文件夹树一致（00、01、02…10、11、99），票据等子文件夹可用 08P（报销）之类带字母序号；
主题：简短中文或英文 slug，不超过 12 字，能独立看懂（如 未决事项清单、defect-cards）；
版本：见 §2（如 v0.3、v1.0-rc、v1.0）；
日期：8 位数字 YYYYMMDD，取本版本定稿日；
示例：02_未决事项清单_v1.0_20260718.md、05_缺陷定义卡_v0.3-rc_20260725.md、08P_火车票_郭小川_860.50_20260720.jpg。
禁止：文件名含空格、含"最终版/最终版2/打死不改版"、含真实客户名、含个人绰号。同一文档只保留一个现行文件，历史版本进 99-归档与原始材料/，不得并排堆放。

3. 脱敏规则（入库前置条件）
客户名替换：一律使用「V公司」（决策 D-01）。禁止出现客户真实公司名称及其简称、logo 文字；发现历史文件含真实名的，先替换再入库，原件移 99-归档与原始材料/ 并登记。
禁真实客户相关人名：V公司侧联系人一律用角色代称（如"V公司工艺主管"），不写真实姓名、电话、微信。
现场图去标识：入库前逐张检查并处理——人脸、工牌、厂牌、含公司标识的工装/设备铭牌、车牌、含坐标信息的门牌一律打码或裁除；保留缺陷与工艺相关画面。去标识后的图片文件名按 §1 命名，并归入 10-现场与工艺资料/（限核心成员）。
工艺参数口径：V公司专属工艺参数按任务书与课程需要引用，禁止外发；空间对外分享默认关闭（见搭建手册 §2）。
归档存疑标注：来源或归属存疑的资料（例如实为V公司复材氧气瓶缠绕工艺的《工艺设计工作总结》）入库时必须在文首标注存疑说明，禁止静默混用为拉挤工艺依据。
4. 禁止项（红线）
禁止入库个人 Agent 灵魂文件：SOUL.md、AGENTS.md 及同类个人 Agent 配置/记忆文件（决策 D-06）。这类文件属个人工作环境，不进飞书、不进仓库共享目录。
禁止未脱敏原件直接入库：含真实客户名、未去标识现场图、含个人联系方式的原件，一律先脱敏；原件只进 99-归档与原始材料/ 的受限区。
禁止大于 100MB 的原始视频直接入库：原始视频统一存放至组内约定的大容量网盘/移动硬盘，飞书与仓库只放：剪辑后片段（≤100MB）、抽帧图、以及指向原始存放位置的索引说明（含拍摄日期、机位、内容摘要）。
禁止上传与课题无关的个人文件；禁止把 M币核算文件标注任何人民币等值表述（M币与人民币严格分离）。
5. 原始数据 raw-data 版本化约定
目录：仓库 raw-data/，按版本子目录组织：raw-data/v0.1_202607XX/、raw-data/v0.2_202608XX/…
只增不改：已进入某版本的原始图片/标注不修改、不删除；纠错（重拍、重标）产生新版本号，旧版本保留可追溯。
组织维度：版本目录内按 physical_piece_id/批次 分件存放，训练/验证/测试拆分（7:2:1）按件划分、禁止同一件跨集，支撑盲测与逐类指标评测。
清单 manifest：每个版本附 manifest.csv（文件名、physical_piece_id、批次、拍摄面、缺陷标注状态、采集参数版本），并记录对应标注工具与版本（LabelImg / X-AnyLabeling）。
现场四类现象图（黑色块状、线状夹杂、纱头/线状、低对比白线）单独子目录存放，标注"观察样本，非训练真值"。
数据版本与模型版本、实验记录三者互相引用：实验记录中必须写明所用 raw-data 版本号。

范汝杰每周六周会后抽查本周新增文件；违规文件 48h 内整改或移出。
