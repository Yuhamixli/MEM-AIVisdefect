1.3 产品原则
页面必须服务管理动作，对准评价铁律三轴（准确度/效率/成本），不做装饰性图表；KPI 无数据一律灰显并注明「无数据」，禁止用 0 或估算值冒充，防「假达标」（已有既定行为，延续到所有新页）。
每个新页上线前做三轴自查（建议写入评审清单）：这个页面让哪项决策更准确（口径同源、证据可点）？让哪个动作更省时（少开一次会、少翻一层目录）？让哪笔 M币花得更明白（消耗可见、超支早报）？答不上来的模块不排期——以此约束 1~2 人编制的 fe-bi 工作流不做表面工程。

3.1 口令方案（二选一，建议）
方案 A（延续现机制）：单一 WRITE_PASSWORD 队入口令，意见箱解锁与一切写回共用；口令只放 Cloudflare Pages secrets，禁止写入 git；本地 dev:api（127.0.0.1:8788）默认口令 dev-password 仅本机使用。
方案 B：意见箱独立口令（如 FEEDBACK_PASSWORD）与队入口令分离；队员/导师用前者，写仓库操作用后者，泄露面更小。
建议：本期选 A——13 人规模单口令足够、运营成本低，导师评论走同一意见箱；若结题后对第三方开放体验再升级 B。状态：建议，待程昱涵与范汝杰确认。
