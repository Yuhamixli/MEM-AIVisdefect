1. 输出目录结构约定
detect-output/
└── {date}/                    # 推理运行日，YYYYMMDD，例 20260728
    └── {piece_id}/            # 样件目录，命名见 §2
        ├── image.jpg          # 检测图（单面场景固定名）
        └── result.json        # 该面检测结果（单面场景固定名）
单面场景（本期默认）：一件一目录，image.jpg + result.json。
多面扩展（360° 布置冻结后启用，关联 OD-1.3）：文件名改为 {surface}.jpg + {surface}.json，如 top.jpg / top.json；surface 取值暂定 top/bottom/left/right，冻结前以字符串透传、前端不做二次推断。
算法侧只新增目录与文件，不改写历史目录；复检产生新 {date} 目录。

3. 文件命名规则
piece_id：P-YYYYMMDD-NNN（样件编号，例 P-20260728-003）；batch_id：B-NN。
文件名只允许小写字母、数字、-、_；图像统一 .jpg（不用 .jpeg/.png 混用）。
JSON 一律 UTF-8 无 BOM；导出一行或美化均可，前端按 JSON 解析不依赖格式。
单张 jpg 质量 85、体积 ≤2MB；超限先在算法侧压缩再落盘。
4. 前端读取方式
路径 A（默认，建议）：新增 npm run sync-results（镜像 sync-data 做法），把 detect-output/ 快照转换+拷入 apps/detector-ui/public/data/detect/，生成 jobs-index.json 与逐件 JSON（文档 22 组三格式）。算法侧只按 §1 落盘，前端零耦合。
路径 B（开发期本地直读）：把样例目录软链或拷入 public/data/detect/，npm run dev 直接读取，供任务 ④⑤ 的 mock 阶段使用。
铁律：detector-ui 只读 public 快照，不直连算法进程、不读训练目录（对齐文档 19 §4.2）。
4.1 样例数据要求（算法侧注入前自检）
样例 ≥2 件：至少 1 件含缺陷、建议另含 1 件 OK 件（defects: []），用于验证「无检出」空态而非白屏；
image_size 与 jpg 实际像素一致（算法侧用 PIL/cv2 读一次校验，不手填）；
model_version 可追溯：能对应到权重文件与训练/推理 commit；
首件样例建议即为本单 §2 示例的实测版（1 裂纹 + 1 气泡），便于与前端渲染逐字段比对；
样例图像为脱敏图：不含 V公司真实产线编号、人员与工装标识（对齐 NFR-FE4 脱敏铁律）。
4.2 联调时间与环境

6. 常见坑（血泪清单）
坐标系原点：bbox＝[x, y, w, h] 像素、原点左上；YOLO 原生输出是归一化中心点格式，换算只在算法导出脚本做一次（乘 image_size），前端不再换算来源格式。
图像缩放比例：显示尺寸≠原图像素；框与掩码一律以 image_size 为基准换算，禁止按显示像素写死。
slug 必须七类之一：标注期示例类（如「毛刺/裂开」）先在算法侧映射表归并到七类 slug，前端收到未知 slug 一律灰显并报警，不透传。
时间格式：一律 ISO 8601 带时区（2026-07-28T14:02:11+08:00），禁止 2026/7/28 下午2点 类字符串。
中文乱码：JSON/md 一律 UTF-8（无 BOM）；CSV 导出加 BOM 防 Excel 乱码；Windows 脚本显式指定 encoding="utf-8"，勿用系统默认 gbk。
置信度越界：confidence 超出 [0,1] 或为 NaN 时，算法导出前 assert 拦截，前端按缺失灰显。
复核状态边界：review_status 默认 pending；confirmed/rejected 是人的复核动作，算法输出不预设。
件级与缺陷级状态一致性：件级 review_status 由缺陷级聚合得出（全部 confirmed 才 confirmed；任一 rejected 即 rejected；否则 pending），聚合规则只在前端实现一处，算法侧落盘一律 pending，两侧不重复实现。
大图加载性能：3200×1920 原图直接渲染会拖慢列表页，列表页用缩略图或懒加载；详情页再加载原图，避免一次请求多张原图卡死演示现场。
6.1 问题记录与升级
联调中发现的问题按以下格式当天记录进 docs/feedback-inbox/inbox/（类别 data）或仓库 issue：现象 + 截图/文件 + 责任侧（算法/前端）+ 修复 commit。单日无法收敛的格式分歧，48 小时内升级范汝杰裁决；涉及 schema 语义变更的，必须先改本单 §2 与文档 22 再改代码（契约先行，顺序不可逆，对齐文档 22 §5 纪律）。
7. 与文档 22 的格式映射
算法落盘格式（本单 result.json）≠ 前端消费格式（文档 22 jobs/{piece_id}.json，faces[] 结构）。sync-results 为唯一转换入口：单面 result.json → faces[1]；多面 {surface}.json 合并为 faces[]；同时汇总生成 jobs-index.json。两侧都不得绕过转换脚本私改格式；schema 语义变更须升 schema_version 并同步改文档 22。
