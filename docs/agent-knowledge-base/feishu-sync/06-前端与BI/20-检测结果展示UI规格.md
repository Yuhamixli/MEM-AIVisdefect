5. 复核工作流
状态机：pending → confirmed / pending → rejected；误判可回退 → pending（记录操作历史）。


要素

约定

状态

操作入口

详情页 ReviewBar；复核队列批量确认/驳回

建议

回写内容

review_status + reviewer + review_time + review_note + 可选 override_class（改判类别）

建议

回写机制（二选一）

A：Pages Function 写 review-overrides/<job_id>.json 回 git（复用意见箱 WRITE_PASSWORD 口令机制）；B：导出复核 CSV，线下合并回 results 目录

建议

本期取舍

先 B 后 A：B 无后端依赖、P1 不挡验收；A 作为增强

建议

权限

算法（黄崇发等）与 PM 可复核；导师只读不可复核

建议

原则

模型输出不直接下 OK/NG 结论；OK/NG 由复核结论给出（单件报告据此表述）

建议
复核队列排序规则（建议）：低置信置顶（按 min(confidence) 升序）→ 按时间正序；已复核条目沉底并可折叠。批量操作仅对当前筛选结果生效，操作前弹确认清单（条数 + 目标状态），防止误批量。改判类别（override_class）只在复核层记录，列表同时显示「原判 → 改判」两个类名并配差异色提示，便于回溯模型混淆模式（如划伤 vs 裂纹），反哺定义卡的易混淆规则修订。每次复核追加一条操作历史（谁、何时、从何状态到何状态、备注），历史只增不改，满足批次追溯叙事（NFR-3）。
6. 空态 / 异常态（必须全部覆盖）


场景

界面表现

无任务 / 检索无结果

灰显「无数据」，给清除筛选按钮；禁止用占位假数据

图像缺失（image_path 404）

图像区灰块注明「图像缺失」；缺陷列表仍可读（坐标照常显示）

JSON 损坏（解析失败）

列表行标「数据损坏」并跳过，不白屏；详情页显示报错摘要（文件路径+解析错误首行）

字段缺失（如缺 model_version）

该字段显示「—」，其余照常

置信度低于阈值

行高亮 + ⚠ 角标，自动进复核队列顶部；阈值建议默认 0.70（软），配置于 detector-ui.config.json，不硬编码

掩码缺失

掩码开关置灰并提示「本任务无掩码」

复核未写回

状态保持 pending，导出时单列注明「未复核」

7. 图像叠加规则


项

约定

框 rect

默认开；线宽 2px（100% 缩放基准）；选中 4px + 角标类别名

掩码 mask

默认关；存在时可开，50% 透明填充 + 同色描边

坐标系

原图像素坐标；显示取整；缩放/平移同步变换

七类固定色板（建议）

8. 导出


格式

内容

用途

JSON

选中范围原样导出（含 review_status 与复核层合并结果）

归档、与算法侧比对

CSV

一行一个缺陷实例：job_id, sample_id, batch_id, face, class_slug, class_zh, severity, x, y, w, h, confidence, model_version, inference_time, review_status, reviewer, review_note

Excel / SPC 统计

单件报告

单页打印友好（A4）：件号头 + 叠加图 + 缺陷表 + 复核结论（OK/NG 依复核）

结题证据、第三方试操演示
导出范围：当前筛选结果 / 单件 / 全批次。文件名建议 export_<scope>_<yyyymmdd-hhmm>.<ext>。导出物是结题证据链的一环：单件报告随 50 件评测归档，CSV 供 quality 工作流做逐类统计，JSON 快照随仓库版本化，保证评审时「看到的界面」与「交出的数据」同一份。

9. 数据规模假设与性能（建议）
规模基线：50 件金标准 × 4 检测面 ≈ 200 个 job；每 job 缺陷实例 0~10 个；单 run 的 index.json 约数百 KB，全量加载无压力，不必做虚拟滚动（保留为优化项）。
原图 3200×1920 量级（采集方案过程版）：图像区默认「适应窗口」缩放显示，叠加层按缩放比变换坐标；建议懒加载当前详情页用图，列表页不预载原图。
离线运行：全部数据随包（public/results/），本地静态服务即可跑；无数据时按 §6 灰显，不出现跨域/外网请求失败弹窗。
9.1 与评测口径的关系（边界声明）
本 UI 只回放与复核，不计算查全率/准确率等验收指标；逐类指标、Wilson 置信区间、按件/按实例判定粒度属 quality 工作流的 50 件评测协议范畴（见需求 G2/G3 与《15·风险登记册》《WBS》相关口径）。页面可放「评测报告」外链入口（建议），但不在前端做指标运算，避免两套口径并存造成「假达标」。

10. 验收口径（给湛佳谕）
七类色板与 §2.3 映射逐类核对一致；框/掩码开关独立生效。
构造损坏 JSON、缺失图像、低置信三类异常样本，界面表现符合 §6 且控制台无未捕获异常。
复核操作只写 override 层；重新载入后检测原值逐字段不变。
CSV 导出列序与 §8 完全一致；单件报告可打印成一页。
全部空态显示「无数据」而非 0 或空白。
11. 未决挂接
OD-COM-01（粗检/精检两阶段一致性待关单）：若最终粗精检并存，job 需加 stage 字段并在列表增列；当前按单层离线结果设计，关单后按变更控制增补。
检测面命名（A/B/C/D 或 上/下/左/右）：建议待 struct-eng 确认后锁定枚举。
置信度阈值 0.70 为软默认：随 50 件评测口径（quality 工作流）标定后回写配置。
