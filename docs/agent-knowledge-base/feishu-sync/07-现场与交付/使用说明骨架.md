1. 文档说明
本说明教你在自己电脑上运行「拉挤制品表面缺陷离线检测模块」：把拍好的电芯压条表面照片放进去，电脑自动找出裂纹、气泡等七类表面缺陷，并给出每件 OK / NG / REVIEW 的初步结论。
读者画像：质检员、工艺员；不需要任何编程或 AI 基础，会复制文件、会按步骤在黑色命令行窗口粘贴命令即可。
章节地图：第 2 章装环境（只做一次）→ 第 3 章 10 分钟跑通样例 → 第 4~5 章日常操作 → 第 6~7 章看结果与复核 → 第 8 章出问题查这里。
约定符号：等宽字体=需要原样输入的命令；【图占位】=正式版此处有截图。
本章验收：读者能口头回答「这工具是干什么的、我能用它做什么、它不能做什么（第 9 章）」。
2. 环境与安装
硬件：普通办公电脑即可运行（用 CPU）；若有 NVIDIA 独立显卡会明显更快。自查方法：任务管理器→性能→看有无「GPU(NVIDIA)」，或命令行输入 nvidia-smi 有无显卡信息输出。【图占位：任务管理器 GPU 页】
软件清单与步骤（全程约 30 分钟，只做一次）：
安装 Anaconda（官网或镜像站下载，一路下一步）；
打开 Anaconda Prompt，创建环境：conda create -n huaduo python=3.10.11 -y；
激活环境：conda activate huaduo；
用清华镜像安装依赖：pip install torch==2.4.1 torchvision==0.19.1 ultralytics==8.4.90 -i https://pypi.tuna.tsinghua.edu.cn/simple；
验证：python -c "import torch,ultralytics;print(torch.__version__,ultralytics.__version__)"，应打印 2.4.1 与 8.4.90。【图占位：验证成功终端截图】
交付包内容说明：权重文件（.pt）、样例图像、配置模板 detect.yaml 的存放位置。
本章验收：第三方在一台干净电脑上仅按本章步骤完成安装，验证命令输出版本号；卡壳时能在第 8 章 FAQ 找到对应条目。
3. 快速开始（10 分钟跑通样例）
三步：① 把交付包内样例图像目录（约 5 张）放到英文路径，如 D:\demo-images；② 粘贴一条命令：detect --input D:\demo-images --output D:\demo-out --model weights.pt --conf 0.25 --save-overlay；③ 打开输出目录，逐张看 overlay.jpg（带框图）与 result.json（文字结果）。
预期：命令行逐张打印处理进度，结束后输出目录按日期/样件分文件夹。【图占位：运行完成终端 + 输出目录结构】
本章验收：读者不看其他文档，10 分钟内跑通样例，并能指出叠加图上的缺陷框位置与类别。
4. 输入准备
图像规格：jpg 或 png；分辨率以采集方案为准（3200×1920 或 3072×2084，待确认）；单件产品按面分图。
命名规则（建议）：样件编号_面_序号.jpg，如 P-20260810-001_top_1.jpg；样件编号与批次登记表保持一致，禁止重名。
目录组织：一次检测=一个输入文件夹，只放本次要检的图；不要混放其他文件。
拍坏（模糊/过曝/黑屏）的图请先剔除；模块对打不开的图会跳过并记录（见第 6 章错误提示），但混入坏图会拉低效率。
本章验收：读者能独立整理好一个合规输入目录，命名抽查无重名、无中文路径。
5. 操作步骤（分步）
按「准备→运行→查看」分 6 步展开，每步给出动作→预期输出→截图占位：
激活环境：conda activate huaduo → 命令行前缀出现 (huaduo)；
进入交付包目录：cd /d <交付包路径>；
运行检测：粘贴 detect 命令（参数含义见下表）→ 逐张打印进度；
等待完成：CPU 单图约几秒级【实测速度待补，禁止预填】；
打开输出目录：detect-output\日期\样件编号\ 下三个文件；
批量复核：见第 7 章。
常用参数白话表：--conf（把握度门槛，默认 0.25；调高=报得更少更保守，调低=报得更多更敏感，拿不准就用默认）；--device 0（有 NVIDIA 显卡时加速）；--save-overlay（生成带框图，建议常开）。【图占位：各步骤命令行截图】
本章验收：读者按步骤完成一次自供图检测，输出三件（image/result.json/overlay）齐全。
6. 结果解读
JSON 白话对照表（节选，正式版逐字段展开）：piece_id=哪一件；surface=哪一面；defects[]=检出的缺陷列表；class_name=缺陷叫什么（七类中文名）；bbox=缺陷在图上的位置框；confidence=模型把握度（0~1，越接近 1 越确定）；overall_decision=本件结论。
三种结论含义与处置：OK=未检出缺陷，正常流转；NG=检出缺陷且触及拒收规则，隔离待判；REVIEW=检出缺陷、需人工复核定夺——初版默认「有任一缺陷即 REVIEW」，全部 REVIEW 件必须人工看图确认。
叠加图怎么看：框的颜色对应缺陷类别、框旁数字是把握度；一张图多个缺陷各有编号（d-001、d-002…），与 result.json 一一对应。【图占位：标注了框/颜色/数字的 overlay 示例】
本章验收：给读者一份陌生的 result.json，能口头复述「哪件哪面、几个缺陷、分别什么类、系统给的结论是什么、下一步该做什么」。
7. 复核与导出
复核动作：对照 overlay.jpg 逐框确认——认可（confirmed）/ 否认（rejected）/ 改类别；复核结论回写到 result.json 的 summary.review_status（回写方式：随复核工具定稿，前端回写接口为草稿态，见《22-api-contract》/api/review，open）。
导出：按批次汇总导出 CSV（件号/面/各类缺陷数/结论/复核状态），供 SPC 与质量台账使用；导出命名 批次号_日期_summary.csv。
归档：输出目录按日期+批次留存，禁止改名删图（追溯链依赖原始文件）。
本章验收：读者能完成一批 10 件的复核与 CSV 导出，且台账字段与 result.json 对得上。
8. 常见问题 FAQ
pip 安装慢/失败：换清华镜像（命令见 2.4）；公司网络有限制请联系 IT 开 PyPI 白名单。
提示找不到模型文件：检查 --model 路径是否指向交付包内的 .pt 文件；路径不要有中文和空格。
跑得很慢：纯 CPU 属正常；确认有 NVIDIA 显卡则加 --device 0。
GPU 不识别 / CUDA 报错：不影响使用，去掉 --device 用 CPU 跑；需要加速再按第 2 章重装匹配显卡的 torch。
误报太多（把好的报成坏的）：先把 --conf 从 0.25 调到 0.35 试；仍多则记录样例反馈给算法同学，勿自行改其他参数。
漏报（明知有缺陷没报出来）：先核对图像是否清晰、缺陷是否在七类之内（对照《缺陷定义卡》）；确认属七类仍漏报，保留原图反馈。
中文乱码 / 打不开 result.json：请用支持 UTF-8 的编辑器（如 VS Code、记事本另存 UTF-8）；不要用 Excel 直接双击打开 JSON。
输出目录是空的：检查 --input 路径是否正确、目录里是否确有 jpg/png；看命令行末尾的汇总行。
个别图没有 result.json 或提示 DECODE_FAIL：该图已损坏或格式不支持，重拍后放入重新检测。
命令行窗口一闪而过：不要双击脚本，先打开 Anaconda Prompt 再粘贴命令，报错会留在窗口里。
类别叫法和现场不一致：以《缺陷定义卡》七类中文名为准（旧名对照见其附录）。
9. 安全与边界
本模块只做表面外观初步判断，不替代最终放行判定；NG/REVIEW 件必须经人工复核，放行权在质量岗位。
模型只认识训练过的七类缺陷（裂纹/气泡/缺纱/划伤/异物/发白/脏污），对新类型缺陷可能漏报；发现疑似新缺陷类型请按第 8.6 条反馈。
模块离线运行，不连接、不改动产线任何设备与安全回路；请勿在产线工控机上自行安装。
