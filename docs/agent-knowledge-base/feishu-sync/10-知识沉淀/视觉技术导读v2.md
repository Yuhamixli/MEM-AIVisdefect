1. 机器视觉检测在干什么
输入一张（或一串）图像 → 判断有没有缺陷 / 在哪 / 什么类型 / 有多大 → 输出报警或记录。
和手机拍照识猫不同：工业场景要稳、快、可解释、可抽检，还要扛光照和产线速度。我们的对象是宽 30~50mm、长约 2m 的拉挤型材，缺陷尺度 8~10mm，精度要求 5mm（加严）——成像质量决定一切下游指标。

    L4：VLM 预标注 → 人工确认（抽检 10%）→ 训练集版本升级（7:2:1）→ 反哺模型
一句话记忆：在线轻、近线精、离线养。完整决策见仓库 docs/02-tech-research/ADR-001-four-layer-vision-stack.md；系统全图见 ../04-需求架构设计/system-architecture.md。
为什么不能把四层串成产线主路径（必考）
召回相乘，无余量：串行链路总召回=各层召回相乘。两层各 90% → 0.9×0.9=81%，对任务书 80% 查全率门槛只剩 1 个百分点余量；串四层（90%⁴≈65.6%）直接不及格。并联/分段则不存在此惩罚。
时延与成本叠加：DINO、SAM2 都是重模型，每帧串行跑四层，节拍和算力（M币）都扛不住。
故障扩散：任何一层抖动都放大到全链路，无法定位、无法复现。
结论（已决议）：四层能力栈采纳为愿景，部署形态修正为「在线轻量+近线精修+离线飞轮」；重模型只做工具层，P0 指标由离线检测模块一线承载。

5. 光学与标定：DOE 先行的原因（新增）
算法吃的是相机+镜头+光源「喂」出来的图。光照乱、模糊、反光，再强的模型也稳不住。
DOE（实验设计）：光源不凭经验锁定，按五维度做对比实验后冻结——眩光控制、均匀性、多缺陷覆盖、前处理对比度、4 面检查（OD-COM-05）。候选：漫射明场/低角度暗场/多方向频闪；偏振、近红外仅在 RGB 不可分时启用。
穹顶光先验：近年同类研究（UWE 2026，复材表面检测）显示穹顶（漫射）光成像 mAP@50=0.817 最优、蓝光暗场最差（0.672）、偏振收益甚微——这是我们 DOE 的假设排序起点，但最终以本课题实测为准。
标定：OpenCV 棋盘格拍 15–20 张，cv2.calibrateCamera 求内参、undistort 去畸变；尺寸标定把「像素」换算成「毫米」，是 5mm 精度判定的前提。
