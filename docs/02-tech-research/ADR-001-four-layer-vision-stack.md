# ADR-001：四层视觉检测技术架构评估

**状态**: Accepted with modification（方向采纳，部署形态修正）  
**日期**: 2026-07-11  
**决策者**: 项目组（待张老师确认）  

---

## 1. 提议方案

| 层 | 职责 | 候选技术 |
|----|------|----------|
| L1 | 异常筛查（产线无监督） | PatchCore |
| L2 | 缺陷定位 + 类型 | Grounding DINO 等开放词汇检测 |
| L3 | 缺陷分割（框不住的难例） | SAM2 |
| L4 | 智能标注 | VLM |

---

## 2. 结论（先看这个）

**方向合理，尤其适合「暂无标注数据」的冷启动；但不能把四层原样串成产线在线主路径。**

建议改为：

```
┌─────────────────────────────────────────────────────────┐
│  在线路径（节拍约束）                                      │
│  图像 → L1 异常筛查 → [可选轻量分类/蒸馏YOLO] → 报警/记录   │
└─────────────────────────────────────────────────────────┘
                         │ 异常帧落盘
                         ▼
┌─────────────────────────────────────────────────────────┐
│  近线/离线精修                                            │
│  L2 定位+类型 → L3 SAM2 分割 → 质检复核                    │
└─────────────────────────────────────────────────────────┘
                         │ 复核结果
                         ▼
┌─────────────────────────────────────────────────────────┐
│  数据飞轮（非实时）                                        │
│  L4 VLM 预标注 → 人工确认 → 训练集升级 → 反哺在线模型       │
└─────────────────────────────────────────────────────────┘
```

---

## 3. 分项评估

### L1 PatchCore — 强烈建议保留（冷启动利器）

| 优点 | 风险 |
|------|------|
| 只需良品即可训练，契合「数据暂时还没有」 | 只回答「异不异常」，不给类型 |
| 工业异常检测成熟（anomalib） | 光照/纹理漂移会导致误报 |
| 可先上产线做筛查，积累异常帧 | 大分辨率需切 patch，要调参 |

**判定**: ✅ 作为 Phase 1 主路径正确。

### L2 Grounding DINO — 适合少样本/开放词表，不宜长期唯一在线检测器

| 优点 | 风险 |
|------|------|
| 文本提示可检新缺陷名，标注少时好用 | 推理偏重，难稳过产线节拍 |
| 与 SAM 提示（框）天然衔接 | 细粒度工业缺陷（细裂纹、干斑）开放词汇未必稳 |
| 可做「零样本试探」 | 最终仍需有监督模型冲 DR/FPR |

**判定**: ✅ 作为近线定位 + 冷启动探测；⚠️ 达标阶段应蒸馏/微调到 YOLO/RT-DETR 等轻量检测器上线。

### L3 SAM2 — 分割层合理，但是「按需」不是「每帧必跑」

| 优点 | 风险 |
|------|------|
| 不规则缺陷（纤维暴露、橘皮、干斑）框不准时掩码更有用 | 需要 L2 给点/框提示 |
| 分割结果利于面积/严重度定量与 BI | 算力与延迟明显高于检测框 |
| 演示效果强 | 不是检出率的充分条件 |

**判定**: ✅ 难例与离线精修；❌ 不要默认全量在线。

### L4 VLM 智能标注 — 正确落点是数据飞轮，不是在线判决

| 优点 | 风险 |
|------|------|
| 加速 13 人团队的标注产能 | 幻觉、工业术语不准 |
| 可生成预标注供人工改 | 延迟与成本高，无法扛节拍 |
| 与 BI「标注进度/质量」看板契合 | 需抽检规范（已有 10% 复核要求） |

**判定**: ✅ 离线标注助手；❌ 不作为在线合格判定器。

## 4. 与项目目标的对齐

| 目标 | 四层原样串行 | 在线/离线拆分后 |
|------|--------------|-----------------|
| DR ≥ 95% | 可能，但难调、难复现 | 更清晰：在线冲 DR，离线冲质量 |
| FPR ≤ 5% | L1 误报会放大到后层 | L1 阈值 + 人工/近线复核可控 |
| ≤ 产线节拍 | ❌ 高风险不达标 | ✅ 在线只跑轻量级 |
| 3 个月交付 | 四层全做完风险高 | 分阶段：L1→L2→L3→L4/BI |

---

## 5. 三阶段落地（匹配 3 个月）

| 阶段 | 周次 | 做什么 | 成功标准 |
|------|------|--------|----------|
| A 冷启动 | W1–W3 | 良品/异常入库；PatchCore 基线；异常帧落盘 | 可演示筛查 + 误报可解释 |
| B 有监督收敛 | W4–W7 | L2 定位/分类；必要时上 YOLO；难例进 SAM2 | mAP@0.5 ≥ 0.7→冲 0.9 |
| C 系统+飞轮+BI | W8–W13 | UI 联动；VLM 预标注；BI（DR/FPR/M币） | M4–M6 演示与验收 |

---

## 6. 决策

1. **采纳**四层能力栈作为系统愿景。  
2. **修正**部署形态为「在线轻量 + 近线精修 + 离线 VLM 飞轮」。  
3. **明确** Grounding DINO 是过渡/近线，最终在线检测器以可满足节拍的模型为准（YOLO 族优先候选）。  
4. **将** L4 与 BI 绑定：标注效率、缺陷分布、实验指标进同一看板。

---

## 7. 后果

- 需在需求中区分在线/离线接口（已写入 `requirements.md`）。  
- GPU 预算要按「在线推理卡 + 离线训练/VLM 卡」分开估（M币核算）。  
- 若张老师要求「纯大模型路线演示」，可做离线全链路 Demo，同时保留轻量在线路径作为工程底线。
