# 团队工作流 / 职能分工

**状态**: 角色未分配；本表为职能工作流建议，非人员名册  
**结构化数据（BI 源）**: [`.project-spec/workstreams.json`](../../.project-spec/workstreams.json)  
**相关**: [team-roster.md](./team-roster.md) · [team-capability-profile.md](./team-capability-profile.md)

> BI「团队」页读取 `workstreams.json` 展示下表。名册与能力表回收后，按能力画像初分到各工作流；**不自动改分工**。

## 原则

1. **单人主职能 ≤1，辅职能 ≤1**；周投入 &lt;4h 不进关键路径 owner。
2. **关键路径**工作流缺人 → 里程碑风险升高，优先补人或降范围。
3. 编制合计可兼职叠加；13 人编制 ≠ 13 条独立全职线。

## struct-eng · 结构/工艺工程师

电芯压条为申报应用对象；外观自动化需与工艺与质量体系对接，本工作流补齐「产品/特性」侧输入。

| 项 | 说明 |
|----|------|
| **ID** | `struct-eng` |
| **建议人数** | 1–2 |
| **关键路径** | **是**（缺人则定义卡允收、APQP 映射与采集关注面易漂） |
| **能力维度** | 建议 `C_DOM`；可兼质量/数据辅职能 |

### 职责

1. 说明电芯压条几何、关注面与外观失效模式。  
2. 将 APQP 控制计划 / 特性清单中的外观项映射到七类缺陷。  
3. 与 `data`/`quality` 共签允收标准，写入缺陷定义卡。  
4. 详设前确认或调整「先三类」名单（默认建议：裂纹、气泡、划伤）。

## agent-ops · Agent 相关工作

本课题用 AI Agent 做项目管理辅助、知识库策展与 BI 数据同步。本工作流是**人机协作面**，不是「把决策外包给模型」。

| 项 | 说明 |
|----|------|
| **ID** | `agent-ops` |
| **建议人数** | 1（常兼；可与 PM / fe-bi / feedback 叠辅） |
| **关键路径** | 否（缺人会拖管理闭环与看板新鲜度，但不直接挡检测里程碑门禁） |
| **能力维度** | 复用 **C_PM + C_FE + C_WR**（暂不新增 `C_AGT` CSV 列；若日后自评需要可再扩展） |

### 职责

1. **PROMPT / 控制面**：维护根目录 `PROMPT.md`、`.cursor/rules` 等 Agent 可读约束，保证会话注入后上下文一致。  
2. **意见箱定时 digest**：按周检索 `docs/feedback-inbox/inbox/`，写 digest、移入 `processed/`；**不**因单条意见自动改需求/章程。  
3. **`.project-spec` ↔ BI sync**：更新结构化 JSON 后跑 `apps/bi` sync，保证 `/team` 等页与文档同源。  
4. **知识库策展**：整理 `docs/agent-knowledge-base/` 与 BI knowledge 目录，保持索引与入门路径可读。  
5. **Agent 规则 / 会话交接**：沉淀规则与 handoff 要点，避免换会话丢状态。  
6. **边界**：Agent **辅助**进度/风险/分工建议；范围、M币超阈、里程碑关闭等**由课题管理者裁决**。

### 典型产出

- 更新后的 `PROMPT.md` / 规则片段  
- 周度意见箱 digest（`docs/feedback-inbox/digests/`）  
- `.project-spec/*.json` 与 `apps/bi/public/data/` 同步记录  
- 知识库条目增补或过期标注  
- 会话交接摘要（状态、未决决策、下一步）

### 周节奏

| 节奏 | 动作 |
|------|------|
| 周一 | 扫进度/风险与未决意见；必要时刷新 `progress-status` |
| 周三（可选） | 小 sync：有文档/JSON 变更则跑 BI sync |
| 周五前 | inbox → digest；核对关键路径工作流是否缺人 |
| 里程碑前后 | 核对 PROMPT 与章程/门禁是否一致；准备 handoff |

与 `feedback` 的分工：意见箱条目收集与催填偏 `feedback`；定时消化、控制面与 BI/知识同步偏 `agent-ops`（可同一人兼）。

## 与角色池的关系

名册中的「角色池」与上表对齐：统筹/PM、结构/工艺、算法检测、算法分割、数据、光学、前端 BI、系统集成为主角色；情报、市场、调研、质量、汇报、意见箱、M币、**Agent 相关工作**多为兼岗或辅职能。

运营模块对照见 [`operation-plan-from-proposal.md`](./operation-plan-from-proposal.md)。

## 变更

- 微调任务归属（不改里程碑）：课题管理者或 PM 支持即可。
- 增删关键路径工作流或改建议人数：走变更控制，并同步更新本文件与 `workstreams.json`。
