## 1. 为什么要做画像

13 人若只按「意向」分工，容易出现：会写前端的全去调模型、没人管 M币与纪要、光学空窗。  
能力画像用来回答三件事：

1. **谁能干什么**（客观技能，不只是想干什么）  
2. **团队缺什么**（角色池缺口）  
3. **任务派给谁**（匹配，而不是平均摊派）

课题管理者据此做角色初分；Agent 在名册+本表回收后可出统计与建议，**不自动改分工**。

## 2. 能力维度（统一量表）

每人按 **0–3** 自评（可请同伴校准一次）：

| 分 | 含义 |
|----|------|
| 0 | 不会 / 未接触 |
| 1 | 了解概念，需带领才能产出 |
| 2 | 能独立完成常规任务 |
| 3 | 能带人、能做关键路径决策 |

### 维度定义

| 代码 | 维度 | 能独立交付的例子 |
|------|------|------------------|
| C_PM | 项目管理 | 周报、风险更新、会议纪要、催办闭环 |
| C_FIN | 成本/M币 | 记账、预算偏差预警、reserve 申请说明 |
| C_DOM | 复材/拉挤领域 | 解释缺陷成因、和工艺同学对话 |
| C_CV | 视觉算法（检测/异常） | 跑通基线实验、读懂 mAP/DR |
| C_SEG | 分割/大模型工具 | SAM/VLM 预标注流程、难例掩码 |
| C_DATA | 数据与标注 | 规范、抽检、版本登记、意见箱数据侧 |
| C_OPT | 光学/硬件 | 相机光源选型对比、成像问题定位 |
| C_FE | 前端/BI | 看板页面、知识页、检测 UI |
| C_SYS | 系统集成 | 联调、演示脚本、环境与发布检查 |
| C_WR | 写作/汇报 | 调研、阶段报告、对张老师材料 |

> **Agent 相关工作（`agent-ops`）**：暂不新增 `C_AGT` 列；匹配时用 **C_PM + C_FE + C_WR**（控制面/同步偏 PM+FE，digest 与策展偏 WR）。若日后自评需要再扩展维度。

## 3. 填表方式

1. 先填 [`team-roster.md`](./team-roster.md) / `team-roster.csv`（基本信息）  
2. 再填 [`team-capability.csv`](./team-capability.csv)（能力分）  
3. 「意向方向」仍以名册为准；画像看的是 **能力分**，两者不一致时以管理者裁决（常见：想做算法但 C_CV=1 → 先配对导师制）

`team-capability.csv` 列：

```text
ID,姓名,C_PM,C_FIN,C_DOM,C_CV,C_SEG,C_DATA,C_OPT,C_FE,C_SYS,C_WR,自评日期,备注
```

## 4. 角色匹配规则（初分用）

| 角色 | 主维度（建议 ≥2） | 辅维度 | 人数建议 |
|------|-------------------|--------|----------|
| 项目统筹 / PM 支持 | C_PM | C_FIN, C_WR | 1 |
| 财务 / M币 | C_FIN | C_PM | 1 |
| 情报与论文 | C_WR | C_DOM, C_CV | 1–2 |
| 行业/竞品调研 | C_WR | C_DOM, C_PM | 1 |
| 需求与变更 | C_PM | C_WR, C_DOM | 1 |
| 算法（检测/异常） | C_CV | C_DATA, C_DOM | 2–3 |
| 算法（分割/标注工具） | C_SEG | C_CV, C_DATA | 1–2 |
| 数据与标注 | C_DATA | C_DOM, C_WR | 2–3 |
| 光学/硬件 | C_OPT | C_SYS | 1–2 |
| 前端 / BI | C_FE | C_SYS | 1–2 |
| 系统集成 | C_SYS | C_FE, C_CV | 1 |
| 质量度量 | C_DATA 或 C_CV | C_WR | 0–1（常作辅） |
| 汇报 / 结题写作 | C_WR | C_PM | 1–2（常兼） |
| 意见箱运营 | C_PM | C_WR | 0–1（常作 PM 辅） |
| Agent 相关工作 | C_PM | C_FE, C_WR | 1（常兼；可与 PM/fe-bi/feedback 叠辅） |

完整工作流、编制矩阵与周节奏见 [`team-workstreams.md`](./team-workstreams.md)。

**负荷规则（管理向）**

- 单人主角色不超过 1 个；辅角色最多 1 个  
- `可投入人时/周 < 4`：不进关键路径 owner，只做支持  
- 某主维度全队 max≤1：记为 **能力缺口**，优先外援/降范围/买服务（记 M币）

## 6. Agent 用法

名册+CSV 回收后，可请求 Agent：

1. 统计各维度人数与缺口  
2. 按匹配规则给出 **2–3 套** 初分方案（标注冲突与风险）  
3. 把缺口写入风险登记册（如「无光学 ≥2」→ 关联 R3）  
4. **不**自动改 `team-roster` 角色列——等课题管理者确认

---

## 7. 隐私与公开

- 能力分默认 **组内可见**（便于协作）  
- 若有人不愿公开细项：可只公开「角色初分结果」，细表仅管理者可见  
- 不对组外（含企业）分享个人评分
