## 1. 项目背景

### 1.1 定义

本项目以**复合材料拉挤成型工艺**中出现的表面缺陷为检测对象。拉挤成型是连续纤维增强复合材料的主要生产工艺之一，广泛应用于风电叶片、轨道交通型材、桥梁加固、电力绝缘材等领域。其表面质量直接影响产品的力学性能、耐候性和客户验收。

### 1.2 典型缺陷类型

| 缺陷类别 | 描述 | 成因 | 严重等级 |
|----------|------|------|----------|
| 裂纹 | 表面沿纤维方向或横向的线状开裂 | 固化应力、温度梯度不当 | 关键 |
| 气泡/气孔 | 表面可见的圆形或椭圆空腔 | 树脂浸润不良、真空度不足 | 关键 |
| 划伤 | 连续机械摩擦痕迹 | 模具表面损伤、脱模不当 | 重要 |
| 纤维暴露 | 表面纤维未被树脂完全覆盖 | 树脂配比异常、浸渍不充分 | 关键 |
| 表面不平/橘皮 | 波状起伏或粗糙表面 | 模具温度波动、拉挤速度不均 | 一般 |
| 色泽不均 | 颜色或光泽度不一致 | 固化程度差异、填料分散不均 | 一般 |
| 白斑/干斑 | 局部树脂缺失导致的白色区域 | 浸渍不良、纤维纱张力不均 | 重要 |

### 1.3 项目目标

- **核心目标**: 开发一套基于机器视觉的在线检测系统，对拉挤制品表面缺陷实现 ≥ 95% 的检出率（DR，Detection Rate）
- **扩展目标**: 缺陷分类准确率 ≥ 90%，误检率（FPR）≤ 5%
- **工程目标**: 单件检测周期 ≤ 产线节拍，支持实时报警与产线联动
- **产出目标**: 技术原型（含算法 + 光学方案 + 软件界面）+ 市场分析 + 成果展示材料

## 4. 技术研究框架（文档 → `docs/02-tech-research/`）

### 4.1 算法路径调研

| 方向 | 代表方法 | 适用场景 | 参考 |
|------|----------|----------|------|
| 传统视觉 | 边缘检测、差分法、阈值分割 | 高对比度缺陷（裂纹、划伤） | OpenCV |
| 通用目标检测 | YOLOv8/v11nn, RT-DETR | 多数缺陷检测 | Ultralytics |
| 异常检测 | PatchCore, EfficientAD, PaDiM | 数据稀少的缺陷类别（仅有良品） | anomalib |
| 语义分割 | U-Net, DeepLabV3+, SegFormer | 需要精确缺陷边界的场景 | mmsegmentation |
| 视觉Transformer | DINOv2, ViT | 小样本场景、无监督预训练 | huggingface |
| 轻量化部署 | YOLO-nano, MobileNet, TinyViT | 边缘端/产线实时检测 | ONNX/TensorRT |

### 4.2 光学方案调研

- **相机选型**: 工业面阵 vs 线扫相机、分辨率、帧率、触发方式
- **镜头选型**: 定焦/变焦、远心镜头 vs 普通工业镜头、工作距离
- **光源方案**: 环形光源、条形光源、同轴光源、背光源——根据缺陷类型选择
- **成像模式**: 明场/暗场/结构光，甚至多光谱
- **运动控制**: 编码器触发、飞拍、线扫同步

### 4.3 系统架构调研

- **采集模块**: 相机 → 采集卡 → 工控机
- **检测模块**: 图像预处理 → 缺陷检测算法 → 缺陷分类定级
- **决策模块**: 报警 → 产线联动（标记/剔除） → 数据记录
- **软件架构**: 客户端程序（C++/Python）+ 可视化界面 + 数据库

## 5. 实施路径（文档 → `docs/04-implementation/`）

### 5.1 阶段划分

```
Phase 1 ──── 基础构建（第1-3周）
  ├─ 数据集采集与清洗
  ├─ 基线算法搭建（YOLO 首版训练）
  └─ 光学平台初选与验证

Phase 2 ──── 算法迭代（第4-7周）
  ├─ 多模型对比实验
  ├─ 数据增强与标注优化
  ├─ 缺陷分类器集成
  └─ 光学方案锁定

Phase 3 ──── 系统集成（第8-10周）
  ├─ 检测系统原型整合
  ├─ UI/可视化开发
  ├─ 产线模拟测试
  └─ 性能调优

Phase 4 ──── 交付冲刺（第10-12周）
  ├─ 产线实测与数据收集
  ├─ 技术文档编写
  ├─ 演示系统搭建
  └─ 结题验收
```

## 8. 补充：你还缺什么（我帮你补全的）

### 8.1 数据集管理计划（`docs/data-management/`）← 通常被忽视的关键项

```yaml
数据采集:
  - 来源: 合作企业产线实拍 / 实验室模拟 / 开源数据集
  - 规模: 初始建议 ≥ 2000 张（含缺陷与良品）
  - 分辨率: ≥ 1920×1080，保证缺陷最小像素覆盖
  - 多样性: 不同光照、不同角度、不同拉挤速度产品

数据标注:
  - 工具: CVAT / LabelImg / Roboflow
  - 格式: COCO JSON / YOLO TXT / VOC XML
  - 标注粒度: 矩形框（检测） + 可选的实例掩码（分割）
  - 多人标注: 需设计标注规范文件，避免标注不一致
  - 质量抽检: 标注后按 10% 比例人工复核

数据版本管理:
  - 建议工具: DVC / HuggingFace Datasets / 自定义版本表
  - 每次实验固定数据集版本号，保证可复现

数据增强:
  - 基础增强: 翻转、旋转、亮度变化、噪声
  - 高级增强: Mosaic、MixUp（YOLO标配）
  - 模拟增强: 针对有限缺陷类型的合成数据（Sim2Real）
```

### 8.2 实验追踪体系（`docs/experiment-tracking/`）

> 使用 MLflow / WandB / TensorBoard 中的至少一种。不要只用手工记录。

每条实验记录必须包含：

```
实验ID | 日期 | 模型 | 数据集版本 | 超参数 | mAP@0.5 | DR | FPR | FPS | 备注
```

建议在 `scripts/` 中预置 MLflow 实验启动模板。

### 8.3 质量度量体系（`docs/quality/`）

除检出率、误检率外，以下补充指标也很重要：

- **缺陷级精度**：不同类型缺陷各自的 Precision/Recall（有些缺陷好检，有些差很多）
- **分类混淆矩阵**：正确分类 vs 误分类的分布
- **边缘情况表现**：弱光、过曝、运动模糊下的退化程度
- **一致性评估**：同一产品多次检测结果是否一致
- **码盘影响**：产线速度对检测精度的影响曲线

### 8.4 知识产权与专利布局（`docs/annex/ip-strategy.md`）

- **可申请专利的方向**：检测方法、光学装置结构、缺陷分类算法
- **软件著作权**：检测系统软件可申请软著
- **论文转化**：算法创新点可撰写学术论文
- **数据知识产权**：标注数据集可登记数据知识产权

### 8.5 风险管理登记册（`docs/risk-management/`）

## 10. 初始任务序列

首次加载本 PROMPT 后，Agent 应该按顺序执行以下动作：

1. **【检查】** 读取 `.project-spec/project-charter.json`，确认项目基本信息是否已填写
2. **【初始化】** 若项目章程为空，按当前项目背景补全基本信息
3. **【盘活】** 读取 `docs/` 下现有文档，盘点当前进度（仅首次加载需要初始化，后续续跑直接进入对应阶段）
4. **【周知】** 输出当前项目状态摘要：阶段位置、已完成/进行中/待办事项
5. **【建议】** 基于当前状态，提出至少一个可供用户确认的下一步具体动作
